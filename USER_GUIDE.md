# Poker EV Calculator 使用说明

## 1. 工具用途

Poker EV Calculator 用于计算德州扑克当前单个决策节点的：

- 手牌权益（Equity）
- 胜率与平分底池概率
- `fold`、`check/call`、`bet/raise` 的净 EV
- 推荐动作
- 基于动作 EV 的近似混合策略

该工具不是完整博弈树 GTO Solver。它使用蒙特卡洛模拟估算当前节点的权益，并结合输入的底池、下注和对手策略计算动作 EV。

## 2. 启动方式

需要 Python 3.10 或更高版本。

浏览器图形界面：

```powershell
python .\web_app.py
```

然后访问：

```text
http://127.0.0.1:8000/
```

命令行界面：

```powershell
python .\gto_ev.py --players 2 --position BTN --hand "As Kh"
```

## 3. 扑克牌输入格式

每张牌使用两个字符表示：

```text
点数 + 花色
```

点数：

| 字符 | 含义 |
|---|---|
| `2` - `9` | 数字牌 |
| `T` | 10 |
| `J` | J |
| `Q` | Q |
| `K` | K |
| `A` | A |

花色：

| 字符 | 含义 |
|---|---|
| `s` | 黑桃 Spades |
| `h` | 红桃 Hearts |
| `d` | 方块 Diamonds |
| `c` | 梅花 Clubs |

示例：

```text
As Kh
```

表示黑桃 A、红桃 K。

公共牌示例：

```text
Qs Jh 2c
```

牌之间必须使用空格分隔。手牌必须输入两张；公共牌可以为空，或输入 3、4、5 张。

## 4. 决策节点参数

### 4.1 牌局人数 `players`

当前仍在牌局中的总人数，包括 Hero。

允许范围：

```text
2 - 9
```

示例：

- 单挑底池：`2`
- 6 人桌且所有玩家仍参与模拟：`6`

人数越多，Hero 赢得整个底池的概率通常越低。

### 4.2 Hero 位置 `position`

支持的位置：

| 位置 | 含义 |
|---|---|
| `UTG` | 枪口位 |
| `UTG+1` | 枪口位后一位 |
| `MP` | 中间位 |
| `HJ` | Hijack |
| `CO` | Cutoff |
| `BTN` | 按钮位 |
| `SB` | 小盲位 |
| `BB` | 大盲位 |

位置用于在没有手动输入对手范围时生成默认范围。后位默认范围通常更宽。

### 4.3 Hero 手牌 `hand`

输入 Hero 的两张底牌。

示例：

```text
Ah Qh
```

### 4.4 公共牌 `board`

输入当前已发出的公共牌：

- 翻牌前：留空
- 翻牌圈：3 张
- 转牌圈：4 张
- 河牌圈：5 张

示例：

```text
Jh 8h 2c
```

## 5. 对手策略参数

### 5.1 策略预设 `strategy`

支持三种对手策略预设：

| 策略 | 含义 | 默认特点 |
|---|---|---|
| `tight` | 紧手 | 入池范围较窄，面对加注弃牌率较高 |
| `balanced` | 平衡 | 使用中等范围和弃牌率 |
| `loose` | 松手 | 入池范围较宽，面对加注弃牌率较低 |

预设主要影响：

- 默认对手范围
- 默认面对加注弃牌率
- 混合策略输出的平滑程度

策略预设不代表真实完整 GTO 范围。如果已知对手数据，应使用高级参数覆盖默认值。

### 5.2 对手范围 `opponent_range`

表示对手可能持有的起手牌范围比例。

输入范围：

```text
0.01 - 1.00
```

示例：

| 输入 | 含义 |
|---|---|
| `0.10` | 最强约 10% 起手牌 |
| `0.25` | 最强约 25% 起手牌 |
| `0.50` | 最强约 50% 起手牌 |
| `1.00` | 任意两张牌 |

范围越窄，对手平均牌力通常越强。

浏览器界面中选择“使用自动范围”时，会根据 Hero 位置与策略预设估算范围。

### 5.3 面对加注弃牌率 `fold_to_raise`

表示 Hero 下注或加注后，对手全部弃牌的概率。

输入范围：

```text
0.00 - 1.00
```

示例：

```text
0.42
```

表示约 42% 概率直接弃牌。

多人底池中，该值被视为所有剩余对手均弃牌的聚合概率，而不是每位对手各自的弃牌率。

## 6. 底池与动作参数

所有筹码值应使用同一单位。推荐统一使用大盲 `BB`。

### 6.1 行动前底池 `pot`

Hero 行动前，桌面上已有的总底池。

示例：

```text
10
```

表示当前底池为 10 BB。

### 6.2 跟注所需 `to_call`

Hero 为继续游戏需要投入的筹码。

示例：

```text
2
```

表示 Hero 需要跟注 2 BB。

特殊情况：

- `to_call > 0`：工具计算 `fold / call / raise`
- `to_call = 0`：工具计算 `fold / check / bet`

当可以免费过牌时，`fold` 通常没有实际意义，但仍作为 EV 基准显示为 0。

### 6.3 加注总额 `raise_size`

Hero 执行下注或加注时，从当前决策点开始投入的总额。

示例：

```text
8
```

如果当前需要跟注 2 BB，输入 `8` 表示 Hero 总共投入 8 BB，而不是在跟注 2 BB 之外额外增加 8 BB。

### 6.4 模拟次数 `simulations`

蒙特卡洛发牌模拟次数。

推荐设置：

| 次数 | 用途 |
|---|---|
| `1000` | 快速预览 |
| `5000` | 日常分析 |
| `10000` | 更稳定结果 |
| `30000` | 高精度分析，耗时更长 |

模拟次数越高，结果通常越稳定，但计算时间也越长。

## 7. EV 与策略解读

### 7.1 权益 `equity`

权益包含直接获胜以及平局时可获得的底池份额。

例如：

```text
权益 45%
```

表示在当前范围与公共牌假设下，Hero 平均可获得约 45% 的最终底池。

### 7.2 动作 EV

EV 是从当前决策点开始计算的净筹码期望。

示例：

| 动作 | EV |
|---|---|
| fold | `0.00 BB` |
| call | `+1.25 BB` |
| raise | `+2.10 BB` |

表示：

- 弃牌作为基准，净 EV 为 0
- 跟注平均盈利 1.25 BB
- 加注平均盈利 2.10 BB
- 当前输入条件下推荐加注

最高 EV 动作会被标记为推荐动作。

### 7.3 近似混合策略

混合策略将各动作 EV 转换为动作频率建议。

示例：

```text
fold   5%
call  20%
raise 75%
```

这不是通过完整 CFR 博弈树求解得到的严格 GTO 频率，而是根据当前节点动作 EV 生成的近似频率。

### 7.4 策略脚本

计算后会生成类似以下内容：

```text
WHEN players=2 AND position=BTN AND hand="As Kh" AND board="Qs Jh 2c"
THEN RAISE

MIX:
  FOLD    5.00%  EV=+0.0000 BB
  CALL   20.00%  EV=+1.2500 BB
  RAISE  75.00%  EV=+2.1000 BB
```

策略脚本用于记录和学习，不会自动操作扑克客户端。

### 7.5 推荐加注范围

计算结果会额外给出当前节点建议用于下注或加注的全部手牌比例：

```text
总加注范围：43.0%
价值范围：26.1%
诈唬范围：16.9%
价值 : 诈唬 = 1.55 : 1
推荐投入额：6.74 BB
```

含义：

- **总加注范围**：建议使用全部可能手牌中的多少比例执行下注或加注。
- **价值范围**：预计被更差牌跟注时仍可获利的加注部分。
- **诈唬范围**：主要依靠对手弃牌获利的加注部分。
- **价值 : 诈唬**：价值组合与诈唬组合的近似比例。
- **推荐投入额**：结合当前底池、跟注额、街道、人数和弃牌率估算的下注或加注总投入。

推荐加注范围会综合考虑：

- Hero 位置
- 当前仍在牌局中的人数
- 翻牌前、翻牌、转牌或河牌街道
- 对手范围宽度
- 对手面对加注的弃牌率
- 紧手、平衡或松手策略预设

该范围是透明的启发式建议，不是完整 Solver 生成的具体手牌矩阵。人数增加、街道变晚或对手范围变强时，推荐范围通常会收紧；对手弃牌率提高时，诈唬范围通常会扩大。

## 8. 浏览器界面输入流程

1. 选择牌局人数和 Hero 位置。
2. 输入 Hero 手牌。
3. 如果不是翻牌前，输入公共牌。
4. 选择对手策略预设。
5. 输入行动前底池、跟注所需和加注总额。
6. 根据需要展开高级参数，修改对手范围、弃牌率和模拟次数。
7. 点击“运行决策分析”。
8. 查看推荐动作、权益、动作 EV、混合策略和策略脚本。

## 9. 桌面截图与离线 OCR

### 9.1 使用 Windows 本地 OCR

1. 点击“选择桌面窗口”并选择牌桌窗口，或点击“导入截图 / 状态”。
2. 点击“离线检测当前画面”。
3. Windows 系统 OCR 会尝试读取画面文字。
4. 检查并修正离线字段。
5. 点击“应用识别结果并分析”。

截图只用于本机 OCR 临时处理，识别后临时文件会被删除。

不同扑克客户端的字体、布局和图形差异较大，OCR 结果必须人工检查。

### 9.2 离线快速校准

如果 OCR 没有正确识别牌局，可直接填写：

- 人数
- 位置
- 手牌
- 公共牌
- 底池
- 跟注额
- 加注额

然后点击“应用识别结果并分析”。

### 9.3 导入状态文件

支持导入 `.json` 或 `.txt`。

JSON 示例：

```json
{
  "players": 2,
  "position": "BTN",
  "hand": "As Kh",
  "board": "Qs Jh 2c",
  "pot": 10,
  "to_call": 2,
  "raise_size": 8
}
```

文本示例：

```text
players 2
position BTN
hand As Kh
board Qs Jh 2c
pot 10
call 2
raise 8
```

## 10. 命令行输入示例

### 10.1 翻牌前

```powershell
python .\gto_ev.py `
  --players 6 `
  --position BTN `
  --hand "As Kh" `
  --strategy balanced `
  --pot 10 `
  --to-call 2 `
  --raise-size 8 `
  --simulations 10000
```

### 10.2 翻牌后

```powershell
python .\gto_ev.py `
  --players 2 `
  --position CO `
  --hand "Ah Qh" `
  --board "Jh 8h 2c" `
  --pot 12 `
  --to-call 4 `
  --raise-size 14 `
  --opponent-range 0.25 `
  --fold-to-raise 0.38 `
  --simulations 10000
```

### 10.3 JSON 输出

```powershell
python .\gto_ev.py --players 2 --position BTN --hand "As Ah" --json
```

## 11. 常见输入错误

### 重复牌

错误示例：

```text
hand As Kh
board As Jh 2c
```

`As` 同时出现在手牌和公共牌中，工具会拒绝计算。

### 手牌数量错误

Hero 手牌必须恰好为两张。

### 公共牌数量错误

公共牌只能为：

```text
0、3、4、5 张
```

不能输入 1 张或 2 张公共牌。

### 花色格式错误

必须使用：

```text
s / h / d / c
```

例如使用 `A黑桃` 或 `ASpade` 无法识别。

### 单位不一致

不要同时混用筹码数量与 BB。

错误示例：

- `pot = 1000` 筹码
- `to_call = 2` BB

正确做法是将所有值统一换算为同一单位。

## 12. 使用限制

- 对手范围使用简化的起手牌强度排序，不是专业 Solver 范围矩阵。
- 多人底池使用相同的对手范围进行采样。
- `fold_to_raise` 是聚合弃牌概率。
- 动作树仅包含当前节点的弃牌、被动动作和一个下注/加注尺度。
- 工具适合学习、对比和快速分析，不应被视为完整 GTO 求解结果。
