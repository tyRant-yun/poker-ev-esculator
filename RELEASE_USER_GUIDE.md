# Release 下载者使用说明

本文面向从 GitHub Release 下载 `poker-strategy-workbench-v2.0.1.zip` 的用户，主要说明 Windows 使用流程。

## 1. 下载文件

在 Release 页面下载：

```text
poker-strategy-workbench-v2.0.1.zip
poker-strategy-workbench-v2.0.1.zip.sha256
```

SHA256 文件用于确认 ZIP 下载完整且未被修改。

PowerShell 校验方式：

```powershell
Get-FileHash .\poker-strategy-workbench-v2.0.1.zip -Algorithm SHA256
Get-Content .\poker-strategy-workbench-v2.0.1.zip.sha256
```

比较两处哈希值是否一致。

## 2. 安装 Python

需要 Python 3.10 或更高版本。

检查是否已经安装：

```powershell
python --version
```

如果提示找不到 `python`，安装 Python 时需要启用“Add Python to PATH”。

本程序只使用 Python 标准库，不需要安装第三方依赖，也不需要运行 `pip install`。

## 3. 解压

将 ZIP 完整解压到具有写入权限的普通目录，例如：

```text
C:\PokerStrategyWorkbench
```

不要直接在 ZIP 压缩包预览窗口中运行程序，否则复盘数据库和启动器可能无法正常工作。

建议避免解压到受保护目录，例如 `C:\Program Files`。

## 4. 启动

在解压目录中双击：

```text
启动德州策略工作台.cmd
```

如果中文文件名启动异常，双击：

```text
start_poker_lab.cmd
```

启动器会：

1. 检查本地服务是否已经运行。
2. 后台启动 `web_app.py`。
3. 等待服务就绪。
4. 打开浏览器页面。

正常访问地址：

```text
http://127.0.0.1:8000
```

也可以在解压目录手动运行：

```powershell
python .\web_app.py
```

手动启动时不要关闭该 PowerShell 窗口。关闭窗口会停止本地服务。

macOS 或 Linux 可以在解压目录运行：

```bash
python3 web_app.py
```

然后访问 `http://127.0.0.1:8000`。

## 5. 创建第一手牌局

在“牌局分析”页面：

1. 选择桌型，支持 2 至 9 人。
2. 设置按钮座位、Hero 座位、盲注和前注。
3. 点击 Hero 手牌区域，从牌组中选择两张牌。
4. 为每位玩家设置有效筹码和对手画像。
5. 点击“创建牌局并开始记录”。

创建后 Live Hand 牌桌会显示：

- Hero 手牌。
- 对手隐藏牌背。
- 当前筹码和本街投入。
- 当前行动玩家。
- 底池和公共牌。

## 6. 记录完整牌局

轮到玩家行动时，界面只显示合法动作：

- 弃牌
- 过牌
- 跟注
- 下注
- 加注
- 全下

下注或加注时可以使用滑块、金额输入框和常用下注比例。

当一条街行动结束时：

1. 点击公共牌选择器。
2. 选择翻牌三张牌，或转牌、河牌一张牌。
3. 点击“发出公共牌”。

进入摊牌后，选择仍未公开的对手手牌并完成结算。

## 7. 策略分析与对手范围

轮到 Hero 行动时，可以点击“运行策略分析”。

分析结果包含：

- 推荐行动。
- 候选动作和尺寸。
- 各候选动作 EV。
- 启发式推荐频率。
- 底池、SPR、跟注成本和有效筹码。
- 关键策略原因。

右侧对手范围区域显示每位未知对手的：

- 范围覆盖率。
- 具体组合数。
- 核心牌型。
- 13×13 范围矩阵。

对手范围会随着行动线动态更新。

## 8. 撤销与创建分支

- “撤销”返回上一个牌局状态。
- “创建分支”复制当前状态，适合尝试另一条行动线。
- 分支不会修改原牌局。

## 9. 结束后继续下一手

牌局结束后可以：

- 选择新的 Hero 手牌并继承当前筹码开始下一手。
- 按本手初始筹码和按钮位置重开。
- 回到本手起点。
- 为零筹码玩家输入补筹金额，使其下一手重新入局。

下一手会使用完整的新牌组，上一手手牌和公共牌不会阻挡选牌。

## 10. 使用复盘库

完成牌局会自动保存。

打开顶部“复盘库”：

1. 在左侧选择历史牌局。
2. 使用节点按钮切换牌局状态。
3. 检查当时的公共牌、筹码、行动和对手范围摘要。
4. 使用搜索框按牌局编号或 Hero 手牌搜索。

复盘数据库位于：

```text
data\poker_workbench.db
```

备份时复制整个 `data` 目录。删除该数据库会清空复盘记录。

## 11. 使用范围训练

打开顶部“范围训练”：

1. 选择位置。
2. 选择 RFI、面对开池跟注或 3-Bet 场景。
3. 在 13×13 矩阵中选择目标牌型。
4. 点击“提交并评分”。

结果颜色：

- 绿色：正确选择。
- 红色：多选。
- 金色：漏选。

内置范围用于训练参考，不代表所有现金桌、锦标赛和不同抽水环境的唯一正确策略。

## 12. 数据和隐私

- 应用在本机运行。
- 不需要账号、API Key 或网络服务。
- 牌局数据不会主动上传。
- Release ZIP 不包含其他用户的复盘数据库。
- 本地复盘数据只保存在解压目录下的 `data` 目录。

## 13. 常见问题

### 页面显示但按钮不可用

原因通常是直接打开了 `web/index.html`，但后端服务没有运行。

解决方法：关闭该页面，双击 `start_poker_lab.cmd`，然后访问：

```text
http://127.0.0.1:8000
```

### 提示找不到 Python

确认：

```powershell
python --version
```

如果命令无效，请重新安装 Python 并加入 PATH。

### 8000 端口被占用

手动使用其他端口：

```powershell
python .\web_app.py --port 8001
```

然后访问：

```text
http://127.0.0.1:8001
```

### 如何停止一键启动器启动的后台服务

在 PowerShell 中运行：

```powershell
$connection = Get-NetTCPConnection -LocalPort 8000 -State Listen
Stop-Process -Id $connection.OwningProcess
```

### 复盘库为空

只有完成的牌局会自动进入复盘库。未结束牌局仍只存在于当前服务内存中。

### 浏览器关闭后数据是否丢失

已完成牌局不会丢失，它们保存在 SQLite 数据库中。未完成的当前牌局在服务重启后不会恢复。

### 如何完全清空复盘数据

停止服务后删除：

```text
data\poker_workbench.db
```

## 14. 当前限制

- 当前版本不是完整 GTO Solver。
- 策略频率属于启发式结果。
- 人机对战尚未实现。
- 未完成牌局不会持久化恢复。
- 当前主要针对本机单用户使用，不建议直接暴露到公网。
